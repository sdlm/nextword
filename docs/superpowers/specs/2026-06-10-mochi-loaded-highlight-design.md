# Подсветка загруженных в Mochi слов в TUI

**Дата:** 2026-06-10
**Статус:** утверждён

## Цель

Внутри TUI визуально выделять слова, которые уже загружены в Mochi, чтобы при просмотре списка было сразу видно, что обработано.

## Визуальное решение

Вся текстовая строка загруженного слова окрашивается в **зелёный** цвет. Незагруженные слова рендерятся как сейчас.

Пример (зелёным — `reluctant`, `genuine`):

```
[ ] B1  interm  840  /  13 abandon          - покидать — 60%
[x] B1  interm  841  /  14 reluctant        - неохотный — 80%   <- зелёная
[ ] B1  interm  842  /  15 acknowledge      - признавать — 70%
[x] B1  interm  843  /  16 genuine          - подлинный — 90%   <- зелёная
```

## Источник истины

`data/mochi_state.json` — словарь `{word: card_id}`. Слово считается загруженным,
если оно присутствует среди ключей. Состояние читается **один раз** при открытии
списка (во время работы TUI файл не меняется).

Матчинг — по точному совпадению слова. В БД и в `mochi_state.json` слова в нижнем
регистре, поэтому регистронезависимость не добавляется (YAGNI).

## Изменения (только `nextword/app.py`, новых файлов нет)

1. Константа `_MOCHI_STATE_PATH` — путь к `data/mochi_state.json` (по аналогии с
   `_POSITION_PATH`).
2. Локальный хелпер `_load_mochi_words() -> set[str]` — читает ключи JSON; при
   отсутствии файла или ошибке парсинга возвращает пустой `set` (по образцу
   `_load_position`).
3. `WordListScreen.__init__`: `self._loaded_words = _load_mochi_words()` — загрузка
   один раз на экран.
4. `WordRow.__init__`: новый параметр `loaded: bool = False`, сохраняется в
   `self.loaded`.
5. `WordRow._text()`: если `self.loaded`, итоговая строка оборачивается в
   Rich-разметку `[green]…[/green]`.
6. `WordListScreen.compose()` и `WordListScreen._rebuild_list()`: при создании
   `WordRow` передавать `loaded=w["word"] in self._loaded_words`.

### Почему локальный хелпер, а не импорт `load_state` из `mochi/upload.py`

Модуль `nextword/mochi/upload.py` на верхнем уровне импортирует `dotenv` и
`requests`. Локальный хелпер на ~4 строки оставляет `app.py` самодостаточным и
совпадает со стилем файла (`_load_position`, `_load_theme` уже сделаны так). Это
соответствует принципу «минимум кода».

## Совместимость

- **Чекбокс независим от подсветки:** `loaded` не связан с `checked`. `toggle()`
  перерисовывает строку через `_text()`, и зелёный цвет сохраняется.
- **Курсор:** подсветка активной строки (фон) накладывается поверх зелёного текста;
  текст остаётся читаемым.
- **Rich-разметка:** существующая строка уже содержит экранированный `\[{mark}]`,
  то есть Static уже парсит markup. Обёртка `[green]…[/green]` не добавляет нового
  риска по сравнению с текущим поведением.

## Тестирование

Юнит-тесты без запуска TUI:

1. `WordRow._text()` содержит `[green]` при `loaded=True`.
2. `WordRow._text()` не содержит `[green]` при `loaded=False`.
3. `_load_mochi_words()` возвращает множество ключей из тестового JSON-файла.
4. `_load_mochi_words()` возвращает пустой `set` при отсутствии файла.

## Вне scope (YAGNI)

- Регистронезависимый матчинг.
- Перечитывание `mochi_state.json` во время работы TUI.
- Влияние на экспорт/фильтрацию (только визуальная индикация).
- Настройка цвета/символа.
